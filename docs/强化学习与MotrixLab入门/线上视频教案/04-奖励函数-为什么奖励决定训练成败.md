# 📚 第四章：Reward——学习信号

### 4.1 Reward 的本质
> **Reward 负责把任务目标转成数值信号。强化学习算法不会理解"真实目标"，只会优化 reward。**

### 4.2 Reward 的输入和输出

def _compute_reward(self):

### 4.3 奖励设计六大原则

#### 原则一：目标一致
```python
# ❌ 错误：任务要求抓住物体，只奖励接近
reward = -distance_to_object

# ✅ 正确：接近 + 抓取双重奖励
reward = -distance_to_object + grasp_success * 10.0
```

#### 原则二：信号密度
```python
# ❌ 稀疏奖励：只有成功时给 1，其他全 0
reward = 1.0 if task_success else 0.0

# ✅ 密集奖励：每一步都有反馈
reward = -distance_delta * 5.0      # 靠近了给奖励
       + action_smoothness * 0.1     # 动作平滑给奖励
       + (1.0 if success else 0.0)   # 成功额外奖励
```

#### 原则三：尺度合理
```python
# ❌ 错误：惩罚量级太大，模型学会不动
reward = -100.0 * abs(action)  # 模型会直接不动

# ✅ 正确：量级相当
reward = -0.01 * sum(action**2)  # 适度的平滑惩罚
    + success_bonus              # 成功奖励
```

#### 原则四：防止漏洞
```python
# ❌ 漏洞：奖励末端速度，模型学会抖动
reward = abs(velocity)  # 抖动也能得高分

# ✅ 修复：同时惩罚抖动
reward = abs(velocity) - 0.1 * abs(velocity_delta)
```

#### 原则五：方向明确
```python
# ❌ 错误：奖励项互相打架，模型不知道该学什么
reward = speed * 2.0              # 鼓励高速移动
       - speed * 3.0              # 又强烈惩罚速度
       + success_bonus

# ✅ 正确：明确主目标，辅助项只做约束
reward = progress_to_goal * 5.0   # 主目标：朝目标推进
       - speed_penalty * 0.1      # 辅助约束：避免过快失控
       + success_bonus * 10.0     # 成功额外奖励
```

#### 原则六：可解释
```python
# ❌ 错误：奖励项来源不清，难以判断是否有用
reward = distance_reward
       + random_bonus * 0.7
       - hidden_penalty * 2.0

# ✅ 正确：每一项都对应明确任务目标
reward = -distance_to_target * 2.0     # 目标1：靠近目标
       + task_success * 10.0           # 目标2：完成任务
       - action_smoothness * 0.05      # 目标3：动作更平滑
```

### 4.4 常见奖励结构
```python
def compute_reward(self):
    # 任务奖励
    task_reward = -self.pole_angle ** 2 - self.cart_position ** 2

    # 成功奖励
    success_bonus = 10.0 if self.is_success else 0.0

    # 稳定性奖励
    stability_reward = -0.01 * sum(self.joint_velocities ** 2)

    # 动作惩罚
    action_penalty = -0.01 * sum(self.actions ** 2)

    # 失败惩罚
    failure_penalty = -10.0 if self.is_failed else 0.0

    # 总奖励
    total = (task_reward + success_bonus + stability_reward
             + action_penalty + failure_penalty)

    return total, {
        'task': task_reward,
        'success': success_bonus,
        'stability': stability_reward,
        'action_penalty': action_penalty,
        'failure': failure_penalty
    }
```

### 4.5 加权组合模板
```plaintext
total_reward =
    task_reward          # 任务本身
    + success_bonus      # 成功奖励
    + stability_reward   # 稳定性奖励
    - action_penalty     # 动作惩罚
    - energy_penalty     # 能耗惩罚
    - failure_penalty    # 失败惩罚
```

### 5.3 代码示例：完整的环境定义
```python
# 1. 定义配置
from dataclasses import dataclass
from motrix_envs.registry import envcfg

@envcfg("cartpole")
@dataclass
class CartpoleCfg:
    # 仿真参数
    sim_dt: float = 0.005
    ctrl_dt: float = 0.02
    max_episode_steps: int = 500

    # 任务参数
    goal_position: float = 0.0
    initial_state_range: float = 0.05

    # 奖励权重
    action_cost_weight: float = 0.001
    velocity_cost_weight: float = 0.0
    position_cost_weight: float = 1.0

# 2. 实现环境
from motrix_envs.base import BaseEnv

class CartpoleEnv(BaseEnv):
    def __init__(self, cfg: CartpoleCfg):
        super().__init__(cfg)
        # 加载模型、创建仿真等

    def reset(self):
        # 重置环境，返回初始观测
        obs = self._get_observation()
        return obs

    def step(self, action):
        # 执行动作
        self._apply_action(action)
        self._simulate()

        # 获取观测
        obs = self._get_observation()

        # 计算奖励
        reward, reward_info = self._compute_reward(action)

        # 判断终止
        terminated = self._is_terminated()
        truncated = self._is_truncated()

        return obs, reward, terminated, truncated, reward_info

    def _compute_reward(self, action):
        # 奖励计算
        reward = -self.pole_angle**2 - self.cart_position**2
        reward -= self.cfg.action_cost_weight * sum(action**2)

        info = {
            'pole_angle': self.pole_angle,
            'cart_position': self.cart_position
        }
        return reward, info
```

## 第六章：常见问题排查

### 6.1 完全学不会？

| 检查项 | 可能原因 | 解决方案 |
|:-----:|:--------:|:--------:|
| reward 太稀疏 | 随机探索很难成功 | 加入中间奖励 |
| observation 缺信息 | 看不到目标 | 检查 observation |
| action 没作用 | 控制接口没接好 | 检查 action mapping |
| done 太早 | 一上来就失败 | 放宽终止条件 |

### 6.2 学到奇怪行为？

| 检查项 | 可能原因 | 解决方案 |
|:-----:|:--------:|:--------:|
| 奖励有漏洞 | 可以作弊得分 | 补充约束项 |
| 辅助奖励太高 | 刷辅助项就能高分 | 降低辅助项权重 |
| 动作惩罚太弱 | 抖动也能接受 | 加大惩罚 |

### 6.3 训练不稳定？

| 检查项 | 可能原因 | 解决方案 |
|:-----:|:--------:|:--------:|
| reset 随机太大 | 每次差异大 | 减小随机范围 |
| sim_dt 不合理 | 仿真不稳定 | 调整步长 |
| action scale 太大 | 动作过大 | 限制动作范围 |
| reward 量级波动 | 各 reward 项量级差大 | 归一化 reward |

### 6.4 reward 上升但任务失败？
这是最容易被忽视的问题！
```python
# ❌ 只看总 reward
if total_reward > threshold:
    print("训练成功！")

# ✅ 看 success rate
if success_rate > 0.9:
    print("训练成功！")

# ✅ 看 reward breakdown
print(reward_info['task'])      # 主任务奖励
print(reward_info['auxiliary']) # 辅助奖励
# 如果辅助奖励涨了但 task 奖励没涨，说明在刷辅助项
```

## 第七章：调试技巧

### 7.1 记录 reward 分项
```python
def _compute_reward(self, action):
    rewards = {
        'task': ...,
        'success': ...,
        'action_penalty': ...,
        'energy': ...
    }
    total = sum(rewards.values())
    return total, rewards
```

### 7.2 看行为视频
> **reward 曲线不能替代行为观察！**

每周至少看一次训练过程中的行为视频，检查：
- 是否在执行预期动作？
- 是否有奇怪的"作弊"行为？
- 边界状态是否正常？

### 7.3 做小实验
```plaintext
Step 1: 固定简单初始状态
Step 2: 降低随机性
Step 3: 用短任务验证方向
Step 4: 逐步增加复杂度
```

### 7.4 检查边界状态
很多 bug 出现在边界：
- ✅ 成功边界（刚好完成任务时）
- ✅ 失败边界（刚好失败时）
- ✅ 超时边界（episode 结束时）
- ✅ 重置边界（reset 后第一帧）

## 总结

### 核心原则
> **读环境时，追踪这五条线：action、state、observation、reward、done。**
> **如果这五条线能讲清楚，一个环境基本就读懂了。**

### 记住三个「不」
```plaintext
EnvCfg  不写复杂流程
Env     不硬编码大量实验参数
Reward  不脱离任务目标
```

### 一句话总结
> **强化学习算法优化的是你写出来的 reward，不是你心里想的目标。**

## 附录：配置参数速查表

### 修改目标 → 优先位置

| 修改目标 | 优先位置 |
|:-------:|:--------:|
| episode 长度 | `EnvCfg.max_episode_steps` |
| 机器人模型 | `EnvCfg.model_file` |
| 动作幅度 | `EnvCfg.action_scale` + `Env.action_mapping` |
| 观测内容 | `Env.observation` |
| 奖励权重 | `EnvCfg.reward_config` |
| 奖励公式 | `Reward._compute_reward()` |
| reset 随机性 | `EnvCfg.reset_noise` + `Env.reset()` |
| 终止条件 | `Env._is_terminated()` |
