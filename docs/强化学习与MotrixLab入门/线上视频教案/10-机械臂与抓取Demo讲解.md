# 机械臂与抓取 Demo

在掌握了基础控制任务（如倒立摆、单摆）后，我们将进入强化学习在机器人领域的重要应用场景——**机械臂控制**。

---

## 一、机械臂控制任务

### （1）机械臂与控制任务

机械臂是一种模拟人类手臂动作的机械装置，通常由多个连杆和关节串联组成，并在最末端配备**末端执行器**（End-effector，如夹爪、吸盘）。

以经典的科研级机械臂 Franka Emika Panda 为例：

| 组成部分 | 说明 |
|----------|------|
| 基座（Base） | 固定在地面或工作台上的基准原点 |
| 7 个旋转关节（Joint 1-7） | Joint 1~4 构成肩部和臂部（大范围空间移动）；Joint 5~7 构成手腕部（末端姿态精细调整） |
| 夹爪（Gripper） | 双指平行夹爪，包含左、右两个平移关节（finger_joint1/2） |

MotrixLab 提供了从简单到复杂的机械臂任务环境：

| 任务名 | 目标 | 机器人 | 任务特点 |
|--------|------|--------|----------|
| 双关节到达（DM Reacher） | 控制末端精确到达指定空间坐标点 | 双关节机械臂 | 最基础的机械臂任务，2D 平面运动，无物理接触 |
| 乒乓球颠球（Bounce Ball） | 使用球拍将乒乓球连续向上击打 | 配天 AIR4-560 6轴工业机械臂 | 高动态响应与速度/时序控制 |
| 抓取立方体（Franka Lift Cube） | 抓取方块并举起至指定高度 | Franka 7轴机械臂 | 典型的刚体接触任务，多阶段 |
| 开抽屉（Open Cabinet） | 抓住抽屉把手拉开至指定深度 | RM65 六轴机械臂 | 运动轨迹必须符合物理滑轨约束 |

### （2）机械臂控制任务的 RL 核心难点

**庞大的状态空间**：观测必须包含本体特征（关节角度/角速度）、末端特征（TCP 位置/姿态）、目标物体特征以及相对几何关系。

**多样的动作设计**：

| 控制维度 | 内容 | 说明 |
|----------|------|------|
| 关节空间 vs 笛卡尔空间 | 关节控制 | agent 直接输出每个具体关节的控制指令 |
| | 末端控制 | agent 输出 TCP 的 6 维位姿，通过 IK 解算 |
| 位置、速度 vs 力矩 | 位置控制 | 直接指定目标位置，底层 PD 控制器负责计算力矩 |
| | 速度控制 | 指定各关节或末端的运动速度 |
| | 力矩控制 | 直接输出电机力矩，难度最高但最柔顺 |
| 绝对控制 vs 增量控制 | 绝对控制 | 每次输出全局坐标或绝对角度 |
| | 增量控制 | 每次输出相对于当前状态的微小偏移量 |

**多阶段奖励调优**：机械臂任务通常包含多个存在因果依赖的阶段，使得奖励编写变得复杂。常见问题包括：
- 稀疏奖励失效
- 多阶段奖励权重平衡（阶段转换是训练瓶颈）
- 防止奖励作弊（Reward Hacking）

## 二、抓取 Demo 讲解

### 了解任务

Franka 抓取立方体任务环境基于 Franka 7 自由度机械臂构建，旨在训练机器人在桌面上抓取立方体并将其提升到指定目标位置。

操作目标：接近目标 → 抓取立方体 → 提升立方体 → 精确定位

### 要素分解

| 要素 | 说明 |
|------|------|
| 环境 | 棕色工作台上放置一个立方体和机械臂 |
| agent | Franka 机械臂 |
| 观测 | 关节角度（9 维）+ 关节速度（9 维）+ 立方体当前位姿（9 维）+ 目标位置命令（7 维）+ 上一时刻动作（8 维）= **36 维** |
| 动作 | **本体 7 维旋转关节**（位置控制模式，增量控制）+ **夹爪二元开闭控制**（Sigmoid 映射→伯努利采样） |
| 奖励 | 接近奖励 + 提升奖励 + 目标跟踪奖励（粗略/精细/接近/加成）+ 动作变化率惩罚 + 关节速度平方惩罚 |
| 初始状态 | 机器人位置和速度初始化，立方体位置初始化 |
| 终止条件 | 方块从桌子掉落、关节速度过大、方块速度过大 |
| 域随机化 | 机械臂关节角度加噪声（±0.125 rad），立方体位置随机，目标位置随机 |

### 关键代码要点

**动作设计**：
- 本体 7 个关节：增量位置控制 `new_joint_pos = actions + old_joint_pos`
- 夹爪第 8 维：Sigmoid 映射到概率 p，伯努利采样决定闭合/打开
- 最终 `clip` 到各关节的物理限位

**奖励设计**（分阶段）：
- 前期（< 20000 步）：宽松的惩罚系数（1e-4），鼓励探索
- 后期（>= 20000 步）：收紧惩罚系数（1e-1），让行为更精细
- 接近 → 提升 → 目标跟踪，每个阶段都有对应的连续奖励引导

**终止条件检查**：
```python
def _check_termination(self, state):
    cube_height = self._cube.get_pose(state.data)[:, 2]
    truncated = cube_height < -0.05  # 方块掉落
    joint_vel = self.get_dof_vel(state.data)
    truncated = np.logical_or(truncated, np.abs(joint_vel).max(axis=-1) > 10)
    cube_vel = self._cube.get_linear_velocity(state.data)
    truncated = np.logical_or(truncated, np.abs(cube_vel).max(axis=-1) > 10)
    return truncated
```
