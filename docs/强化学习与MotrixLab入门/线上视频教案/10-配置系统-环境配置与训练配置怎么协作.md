# 配置系统：环境配置与训练配置怎么协作

## EnvCfg 与 RLCfg 的分工

### EnvCfg：环境与仿真

`EnvCfg` 在 `motrix_envs` 里，描述物理仿真与任务本身，和用哪种 RL 算法无关。

```python
# motrix_envs/src/motrix_envs/base.py
@dataclass
class EnvCfg:
    """Config for the environment"""
    model_file: str = None              # 模型文件路径
    sim_dt: float = 0.01                # 物理仿真内部时间步长（秒）
    max_episode_seconds: float = None   # 每个 episode 的最大时长（秒）
    ctrl_dt: float = 0.01               # 控制频率对应的时间间隔（秒）
    render_spacing: float = 1.0         # 渲染间隔
    # ...
```

典型内容：模型文件、仿真/控制步长、单局时长（或步数推导）、渲染间隔；各环境子类还会加任务专属字段（初始姿态、奖励相关几何、域随机化等）。

注册方式：通过 `@envcfg("cartpole")` 等把环境名注册到一个 `EnvCfg` 子类，放进 `motrix_envs.registry`，再挂上具体 `ABEnv` 实现。

---

### RLCfg（`BaseRLCfg` / `PPOCfg`）：训练与算法

在 `motrix_rl` 里，描述怎么训：并行环境数、总步数、存盘间隔、随机种子，以及 PPO 的网络结构、学习率、rollout、clip 等。

```python
# motrix_rl/src/motrix_rl/base.py
@dataclasses.dataclass
class BaseRLCfg:
    """Config for the reinforcement learning algorithm"""
    # Basic training parameters
    seed: Optional[int] = None               # 随机种子
    num_envs: int = 2048                     # 并行运行的环境数量（训练时）
    play_num_envs: int = 16                  # 并行环境数量（评估/演示时）
    max_env_steps: int = 20480000            # 整个训练过程的最大环境步数
    check_point_interval: int = 1000         # 每多少步保存一次模型检查点
    # ...
```

`PPOCfg` 在 `BaseRLCfg` 上继续加策略/价值网络、PPO 超参、奖励缩放等：

```python
@dataclass
class PPOCfg(BaseRLCfg):
    """PPO configuration for SKRL"""
    # Model architecture settings
    policy_hidden_layer_sizes: tuple[int, ...] = (256, 128, 64)
    value_hidden_layer_sizes: tuple[int, ...] = (256, 128, 64)
    share_policy_value_features: bool = True

    # Agent settings
    rollouts: int = 32
    learning_epochs: int = 2
    mini_batches: int = 32
    discount_factor: float = 0.99
    lambda_param: float = 0.95

    # Learning rate
    learning_rate: float = 1e-3
    learning_rate_scheduler_kl_threshold: float = 0.008

    # Training
    random_timesteps: int = 0
    learning_starts: int = 0
    grad_norm_clip: float = 1.0
    time_limit_bootstrap: bool = True

    # PPO clipping
    ratio_clip: float = 0.2
    value_clip: float = 0.2
    clip_predicted_values: bool = True

    # Loss
    entropy_loss: float = 0.2
    value_loss_scale: float = 2.0
    kl_threshold: float = 0

    # Reward shaping
    rewards_shaper_scale: float = 1.0
```

注册方式：`@rlcfg("cartpole")` 把同一环境名下、框架（如 skrl）+ 后端（jax/torch）对应的配置类登记到 `motrix_rl.registry`；`default_rl_cfg` 按环境名和 backend 取出实例。

---

### 二者关系

- **环境配置 (EnvCfg)**：控制物理仿真和任务行为，包括仿真参数、重置噪声、时间限制等
- **训练配置 (RLCfg)**：控制强化学习算法，包括网络结构、学习率、批次大小、训练步数等

---

## 参数继承、覆盖与管理方式

### 继承：用 `@dataclass` 子类叠加默认值

**环境 `EnvCfg`**：在 `EnvCfg` 上继续派生，子类只改差异字段；复杂任务常用嵌套 dataclass（`field(default_factory=...)`）成组管理。

```python
# go1-flat-terrain-walk 的 cfg 示例
@registry.envcfg("go1-flat-terrain-walk")
@dataclass
class Go1WalkNpEnvCfg(EnvCfg):
    max_episode_seconds: float = 20.0
    model_file: str = os.path.dirname(file) + "/xmls/scene_motor_actuator.xml"
    noise_config: NoiseConfig = field(default_factory=NoiseConfig)
    control_config: ControlConfig = field(default_factory=ControlConfig)
    reward_config: RewardConfig = field(default_factory=RewardConfig)
    init_state: InitState = field(default_factory=InitState)
    commands: Commands = field(default_factory=Commands)
    normalization: Normalization = field(default_factory=Normalization)
    asset: Asset = field(default_factory=Asset)
    sensor: Sensor = field(default_factory=Sensor)
    sim_dt: float = 0.01
    ctrl_dt: float = 0.01
```

**训练 `RLCfg`**：`PPOCfg` 继承 `BaseRLCfg`；各任务在 `motrix_rl/cfgs.py` 里再继承 `PPOCfg`，用类体默认值覆盖 PPO 训练字段。

```python
class basic:
    @rlcfg("cartpole")
    @dataclass
    class CartPolePPO(PPOCfg):
        max_env_steps: int = 10_000_000
        check_point_interval: int = 500
        policy_hidden_layer_sizes: tuple[int, ...] = (32, 32)
        value_hidden_layer_sizes: tuple[int, ...] = (32, 32)
        rollouts: int = 32
        learning_epochs: int = 5
        mini_batches: int = 4
        learning_rate: float = 3e-4
```

---

### 注册时覆盖

**EnvCfg**：`motrix_envs.registry.make` 先用注册的 `env_cfg_cls()` 得到默认实例，再用字典逐项 `setattr`：
```python
env_cfg = meta.env_cfg_cls()
if env_cfg_override is not None:
    for key, value in env_cfg_override.items():
        if hasattr(env_cfg, key):
            setattr(env_cfg, key, value)
        else:
            raise ValueError(f"Config class '{env_cfg.class.name}' has no attribute '{key}'")
```

**RLCfg**：`Trainer` 先按环境名 + backend 从 registry 实例化默认的 `PPOCfg`（或子类），再用 `cfg_override` 做不可变式覆盖：
```python
rlcfg = registry.default_rl_cfg(env_name, "skrl", backend="jax")
if cfg_override is not None:
    rlcfg = rlcfg.replace(**cfg_override)
self._rlcfg = rlcfg
```

---

### 运行时覆盖

**环境侧**：`env_registry.make` 里对默认 `meta.env_cfg_cls()` 做 `setattr` 按 key 覆盖；key 必须是该 `EnvCfg` 上存在的属性，否则报错。

当前 `ppo.Trainer.train` 里 `make` 未传入 `env_cfg_override`，所以环境侧运行时 patch 主要靠在调用 `make` 时自己传。

**RL 侧**：`scripts/train.py` 只把少量 flag 写进 `rl_override`（例如 `--num-envs`、`--seed` / `--rand-seed`），再传给 `Trainer(..., cfg_override=rl_override)`。

---

## 如何通过配置灵活控制实验行为

### 不改代码

训练（`scripts/train.py`）可直接影响实验：
- `--env`：换环境名即换一套 EnvCfg 注册类 + RL 注册类
- `--train-backend jax|torch`：走不同后端
- `--sim-backend`：选仿真后端
- `--num-envs`：经 `rl_override` 写入 `PPOCfg.num_envs`
- `--seed` / `--rand-seed`：控制 `rlcfg.seed`
- `--render`：是否可视化训练

### 改配置类、少改逻辑

- 任务 / 物理 / 奖励形状：为不同实验注册不同环境名与 `EnvCfg` 子类
- 算法与算力：在 `motrix_rl/cfgs.py` 里用 `@rlcfg("env-name")` 挂不同 `PPOCfg` 子类
- JAX vs Torch 差异：同一 env 注册两个 backend 专用类，由 `--train-backend` 自动选型

### 程序里动态改

**环境：**
```python
env = registry.make(
    name="my-custom-env",
    sim_backend="np",
    num_envs=256,
    env_cfg_override={"custom_param_1": 2.0, "reset_noise_scale": 0.02}
)
```

**训练：** 构造 `ppo.Trainer(..., cfg_override={...})`，内部会对 `default_rl_cfg` 得到的实例做 `rlcfg.replace(**cfg_override)`。
