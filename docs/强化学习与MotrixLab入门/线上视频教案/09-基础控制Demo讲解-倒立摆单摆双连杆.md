# 基础控制 Demo 讲解：倒立摆 / 单摆 / 双连杆

本节内容讲解基础控制任务：倒立摆 / 单摆 / 双连杆，剖析强化学习中"小规模状态空间与动作空间"的典型设计思路。

在强化学习与机器人控制领域，无论是机器狗灵活运动，还是灵巧手复杂操作，其底层的物理交互逻辑皆根植于经典力学。在迈向高维复杂系统之前，我们先回归基础任务，研究**倒立摆**（CartPole）、**单摆**（Pendulum）和**双连杆**（Acrobot）三个经典的入门案例。

---

## 一、基础控制任务的特点

基础控制任务不涉及复杂的视觉信息（如 RGB 图像、点云）与复杂的机械结构，仅包含最简单的物理结构与纯粹的刚体动力学。整体来讲，基础控制任务具有以下三个共同特点：

1. **低维状态空间**：观测维度通常为 3~6 维，学习速度快
2. **低维动作空间**：动作空间为 1 维连续控制，易于调试
3. **快速收敛**：通常几分钟到十几分钟即可完成训练

---

## 二、任务环境详解

### 倒立摆（CartPole）

**任务描述**：一根杆子通过铰链连接在小车上。智能体需要通过给小车施加左右方向的力，让杆子保持直立不倒。

**预期训练结果**：
- 杆子角度大部分时间保持在 ±5 度以内
- Episode 时长接近或超过 10 秒
- 小车在合理范围内移动以保持平衡

**要素分解**：

| 要素 | 说明 |
|------|------|
| 环境 | 一根紫色横杆作为小车的轨道，一根杆子通过铰链连接在小车上 |
| agent | agent 输出一个浮点数，代表施加在小车水平方向上的力 |
| 观测 | **(4D 连续)**：`Box(-inf, inf, (4,))`。包含：小车位置、杆子角度、小车速度、杆子角速度 |
| 动作 | **(1D 连续)**：`Box(-3.0, 3.0, (1,))`。水平方向上的力 |
| 奖励 | 每坚持 1 步不倒下，获得固定奖励 `+1.0`（存活奖励） |
| 初始状态 | 小车初始位置 0.0m，杆子初始角度 0.0 rad，初始速度均为 0 |
| 终止条件 | 杆子角度超过 ±0.2 rad（~±11.5°），小车位置超出 [-0.8, 0.8]m，或出现 NaN；最大 1000 步 |
| 域随机化 | 初始状态上添加小幅随机噪声（`reset_noise_scale = 0.01`） |

**关键代码要点**：
- 观测：`obs = np.concatenate([dof_pos, dof_vel], axis=-1)` → 4 维向量
- 动作：`state.data.actuator_ctrls = actions` → 直接写入水平力
- 奖励：`reward = np.ones((self._num_envs,), dtype=np.float32)` → 存活奖励
- 终止：角度 > 0.2 rad 或小车位置超出 [-0.8, 0.8]

### 单摆（Pendulum）

**任务描述**：单关节摆起并倒立保持任务。目标是用一个电机扭矩把摆甩起并稳定在倒立位置。

**要素分解**：

| 要素 | 说明 |
|------|------|
| 环境 | 一段杆体和一个铰接关节组成单摆 |
| agent | 铰链的驱动电机 |
| 观测 | **(3D 连续)**：`Box(-inf, inf, (3,))`。包含：`cos(θ)`、`sin(θ)`、角速度 |
| 动作 | **(1D 连续)**：`Box(-1.0, 1.0, (1,))`。代表电机施加的扭矩（归一化后映射到实际力矩范围） |
| 奖励 | 倒立奖励 + 能量 shaping + 惩罚（角速度²、ctrl²、ctrl_delta²） |
| 初始状态 | 角度随机于 `[-π, π]`，控制历史初始化为 0 |
| 终止条件 | 仅 NaN 检查，其他由时长限制 |

**观测中用 cos/sin 的考虑**：因为原始角度在 π 和 π+2kπ 之间存在数值突变，使用三角函数可以保证状态空间的连续性。

**奖励函数可视化**：
```
reward = 3.0 * upright + energy_reward - vel_penalty - 0.001 * ctrl² - 0.001 * ctrl_delta²
```

- `3.0 * upright`：姿态分（权重最大，引导保持垂直）
- `energy_reward`：能量分（引导将能量维持在直立所需水平）
- `vel_penalty`：速度惩罚（防止高速转动）
- `0.001 * ctrl²`：能耗惩罚（鼓励最小力气）
- `0.001 * ctrl_delta²`：平滑度惩罚（惩罚高频震荡）

`tolerance`（容差）函数：将物理世界的连续状态转化为 [0, 1] 之间的平滑奖励信号，避免稀疏奖励导致的学习困难。

### 双连杆（Acrobot）

**任务描述**：Acrobot 是模拟体操运动员在单杠上的动作的任务。这是一个欠驱动系统，有两个可以运动的关节，但**只有肘关节配置了电机**（肩关节是被动的）。智能体需要通过控制肘部电机的扭矩，把双臂甩起，使末端到达目标区域。

**要素分解**：

| 要素 | 说明 |
|------|------|
| 环境 | 两端杆体，一个杆体一端固定于铰链（无动力），另一端的铰链有动力并连接第二个杆体 |
| agent | 安装在肘关节处的电机 |
| 观测 | **(6D 连续)**：上臂与下臂的水平投影（2个）、垂直投影（2个）、肩关节角速度、肘关节角速度 |
| 动作 | **(1D 连续)**：`Box(-1.0, 1.0, (1,))`。仅控制肘关节的扭矩 |
| 奖励 | 基础稀疏奖励 + 持续奖励 + 距离奖励 - 速度惩罚 |
| 初始状态 | 肩关节角度随机于 `[-π, π]`，肘关节角度随机于 `[-π, π]`，角速度初始化为零 |
| 终止条件 | 仅 NaN 检查 |

**奖励设计亮点**：
- `base_rwd`：稀疏奖励（margin=0，阶跃函数，进圈=1，不进=0）
- `distance_reward = 0.3 * (1.0 - clip(dist/2.0, 0, 1.0))`：引导性连续奖励
- `velocity_penalty`：惩罚过高速度
- 用 `distance_reward` 牵引方向，用 `base_rwd` 确认成功，用 `velocity_penalty` 规范行为

## 三、小规模状态空间与动作空间分析

### 三大任务的控制难点与异同点

**共同面临的真实控制难点**：
- **开环不稳定性**：系统在缺乏持续且精准的外部控制时，会迅速偏离目标状态
- **欠驱动特性**：系统控制输入维度少于自由度（倒立摆 2 自由度 1 输入，双连杆 2 自由度 1 输入）
- **非线性动力学**：包含旋转关节的系统涉及重力与向心力的复杂非线性耦合

### 小规模状态空间的设计总结

1. **必须包含"位置"与"速度"**：仅有位置不够，必须同时包含 `dof_pos` 和 `dof_vel`
2. **连续周期量的几何编码**：对于可连续旋转的关节，用 `cos(θ)`、`sin(θ)` 代替原始角度，消除边界跳变

### 小规模动作空间的设计总结

1. **归一化与裁剪**：策略通常输出 [-1.0, 1.0] 的标准区间，环境内部再线性映射到实际物理范围
2. **连续动作 vs 离散动作**：三个环境均采用连续动作空间（Box），允许更平滑、微小的调节力矩

**总结**：基础控制任务虽然结构极简，但其对于状态变量的选择、数值跳变的处理、以及动作的归一化映射，完全可以直接套用在数十个自由度的四足机器狗或人形机器人身上。理解了这三个任务，就相当于掌握了机器人强化学习环境搭建的通用范式。
