# 优秀周报评审标准

本标准适用于训练营理论学习周报和项目实践周报，用于学员提交、自查和助教评审。

> [!IMPORTANT]
> “优秀周报”必须同时满足两项条件：总分不低于 **85 分**，且对应类型的必交证据全部完整。内容写得多、训练最终成功，均不能替代实验与复现证据。

## 一、周报类型

提交前须明确标注周报类型：

- **理论学习周报**：以概念学习、原理推导、代码练习和知识应用为主。
- **项目实践周报**：涉及环境搭建、任务开发、奖励设计、训练、调参、测试或问题排查。

同一周同时包含理论和实践内容时，按项目实践周报评审。

## 二、项目实践周报必交证据

以下材料缺少任意一项，本周报不得评为优秀：

### 1. 目标、完成情况与下周计划

- 写明本周要解决的具体问题和预期结果。
- 对照目标逐项说明已完成、未完成及变更内容。
- 下周计划须包含可检查的任务和预期产出，避免只写“继续学习”“继续调参”。

### 2. 奖励函数截图

- 截图须清晰展示奖励项名称、权重及本周修改。
- 正文须解释主要奖励项的设计目的、修改原因和预期影响。
- 仅粘贴代码文本但无截图，或只有截图但无解释，均不满足该项要求。
- 如本周未修改奖励函数，仍须提供当前奖励函数截图，并明确说明“本周未修改”及原因。

### 3. 最终效果视频

- 视频须连续展示任务环境、机器人行为和最终效果，不能只提供单帧截图。
- 周报中须标注视频对应的实验编号、模型/checkpoint 或代码版本。
- 视频链接须对评审人员可访问，且不得使用与本周实验不对应的历史视频。
- 训练未成功时，可提交当前最佳结果或典型失败视频，但须说明失败现象和后续判断。

### 4. 至少三组实验曲线

至少包含：

1. 一组基线实验；
2. 第一次参数修改实验；
3. 第二次参数修改实验。

曲线须满足：

- 使用相同指标名称和可比较的横纵坐标范围。
- 标注实验编号，能够与实验对比表、配置和结论对应。
- 原则上每次只修改一个主要变量，以验证该变量的影响。
- 如同时修改多个参数，须解释组合修改的原因；否则该组实验不能作为有效归因证据。
- 不得只展示表现最好的一组曲线而隐去失败实验。

建议至少展示总奖励曲线，并根据任务补充成功率、episode length、速度跟踪误差、姿态误差或各奖励分项。

### 5. 实验对比表

实验对比表至少记录：

| 实验编号 | 目的/假设 | 相对上一组的参数变化 | 训练步数 | 随机种子 | 环境数量 | 结果指标 | 结论 |
| --- | --- | --- | ---: | --- | ---: | --- | --- |
| E0 基线 | 建立对照 | 无 |  |  |  |  |  |
| E1 修改一 |  |  |  |  |  |  |  |
| E2 修改二 |  |  |  |  |  |  |  |

无法固定随机种子时须注明原因。不同实验的训练步数或环境数量不一致时，须说明比较方法和限制。

### 6. 关键问题复盘

至少完整复盘一个具体问题，包含：

1. **现象**：观察到了什么异常；
2. **假设**：认为可能由什么导致；
3. **验证**：使用什么日志、代码检查或对照实验验证；
4. **解决方案**：实际修改了什么；
5. **结果**：修改后有什么变化，假设是否成立。

### 7. 复现信息

至少提供：

- 代码仓库、分支及 commit ID；
- 运行环境和关键依赖版本；
- 完整训练与推理命令；
- 主要配置文件或配置链接；
- 日志、模型/checkpoint、曲线和视频位置；
- 必要的硬件信息和训练时长。

## 三、理论学习周报必交证据

理论学习周报不强制提交奖励函数截图、训练曲线和最终视频，但必须包含：

- 学习目标及实际完成情况；
- 核心原理总结或关键推导；
- 代码练习、问题解答或可验证案例；
- 至少一个误区、疑问或知识应用分析；
- 下周计划。

只摘抄课程内容、罗列阅读章节或粘贴 AI 总结，不构成优秀周报。理论周报须体现个人理解，并提供能够验证学习结果的练习或案例。

## 四、评分规则

总分为 100 分。

| 评分维度 | 分值 | 项目实践周重点 | 理论学习周对应要求 |
| --- | ---: | --- | --- |
| 目标与实际完成情况 | 10 | 目标具体，完成状态和偏差说明清楚 | 学习目标、完成范围及未解决问题清楚 |
| 技术方案与奖励设计 | 20 | 环境、观测、动作、奖励或算法方案合理，奖励修改有依据 | 原理理解、推导过程和方法选择正确 |
| 多轮实验及曲线对比 | 25 | 至少三组有效实验，变量、曲线、指标和结论对应 | 多组练习、案例或方案对比，能够验证理解 |
| 最终成果与视频证据 | 15 | 最终视频可验证，并与实验、模型版本对应 | 代码、推导结果、问题解答或可运行案例完整 |
| 问题定位与复盘 | 15 | 现象、假设、验证、方案、结果形成闭环 | 能识别误区或疑问，并完成分析和验证 |
| 可复现性 | 10 | 代码版本、命令、配置、日志和模型位置完整 | 资料来源、代码、输入和运行方法可复核 |
| 表达和提交规范 | 5 | 结构清晰，截图、曲线、表格和链接可读有效 | 结构清晰，引用规范，图表和链接有效 |

### 分档参考

- **优秀表现**：证据完整，结论由数据或案例支持，其他人能够复核关键过程。
- **合格表现**：主要工作和结果清楚，但对照、复现信息或问题分析不充分。
- **待改进**：以工作流水账和主观结论为主，缺少可验证证据。

## 五、优秀认定与否决规则

优秀周报必须同时满足：

- 总分不低于 85 分；
- 对应周报类型的必交证据完整；
- 实验参数、曲线、视频、模型版本和正文结论能够相互对应；
- 所有附件和链接在评审时可正常访问。

出现以下任一情况，不得评为优秀：

- 项目实践周缺少奖励函数截图、最终效果视频或三组有效实验曲线；
- 缺少实验对比表、关键问题复盘或必要复现信息；
- 曲线指标或坐标不可比，却直接得出参数优劣结论；
- 伪造、篡改实验数据，或选择性隐瞒影响结论的失败实验；
- 将未经验证的 AI 内容作为自己的实验事实或技术结论；
- 证据链接失效，且在评审期限内未补交。

训练最终未成功不直接失去评优资格。若实验记录完整、问题分析严谨、结论有效且全部硬门槛满足，仍可获得高分。
